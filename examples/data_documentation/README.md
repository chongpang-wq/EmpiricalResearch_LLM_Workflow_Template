# Example Data Documentation

This folder contains template example files only.

The files demonstrate how raw data and codebooks should be documented in a new empirical research project.

They are not real project data.

## Mandatory Cleanup Rule

During project initialization, after real raw data are added or linked and documented, the coding LLM must delete this entire folder unless the human researcher explicitly approves retaining it as template documentation only.

Do not use these example files for data processing, analysis, testing, intermediate data, final data, tables, or figures.

## Files

| File | Purpose |
|---|---|
| `example_raw_data.csv` | Header-only example showing common fields for a raw-data table. |
| `example_codebook.csv` | Example codebook structure for documenting variables. |

## How to Use This Example

When documenting a real dataset, write a source-specific README that records:

- data source;
- original location;
- access restrictions;
- unit of observation;
- key identifiers;
- time and geographic coverage;
- variable groups;
- monetary units and deflation rules when relevant;
- administrative codes and boundary-change cautions when relevant;
- known caveats;
- codebooks or questionnaires.
