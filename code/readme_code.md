# Code README

This folder stores project code for data processing, empirical analysis, validation, and result generation.

## Core Rules

- Raw data must be read-only.
- Durable data outputs must be generated by code.
- Data-processing code should write to `data_intermediate/` or `data_final/`, not to `code/`.
- Analysis code should write tables and figures to `result/` only after explicit human approval.
- Do not silently change sample restrictions, controls, fixed effects, clustering, estimators, treatment definitions, variable definitions, or output paths.

## Main Code Areas

| Folder | Purpose | README |
|---|---|---|
| `data_process/` | Cleaning, construction, harmonization, merging, validation, and final dataset generation. | `data_process/readme_data_process.md` |
| `empirical_analysis/` | Regressions, mechanisms, robustness checks, tables, figures, and empirical scripts. | `empirical_analysis/readme_empirical_analysis.md` |

## Documentation Requirements

Scripts that create durable outputs should document:

- purpose;
- input files;
- output files;
- unit of observation;
- key identifiers;
- sample restrictions;
- important assumptions;
- validation checks.

## Skills Trigger

When modifying code, consult `skills/short_silent_bug_checklist.md`. Use the full checklist only when the human asks for a comprehensive code audit.
